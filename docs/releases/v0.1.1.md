# Release Notes - v0.1.1

**Release Date:** 13 March 2025
**Type:** Minor Release

## Overview

Version 0.1.1 adds reliability improvements and automatic dependency management for Node.js packages required for markdown conversion.

## ✨ New Features

### Automatic Node.js Package Installation

- **Auto-install:** Automatically installs required Node.js packages
- **Dependencies:** `md-to-adf` package for markdown to Atlassian Document Format conversion
- **User Experience:** Seamless setup without manual npm commands

### HTTP Request Retry and Backoff

**Improved Reliability:**
- Automatic retry logic for failed HTTP requests
- Exponential backoff strategy for rate limiting
- Handles dropped connections gracefully
- Manages transient API errors

**Benefits:**
- More resilient to network issues
- Better handling of Jira API rate limits
- Reduced failures in production environments
- Automatic recovery from temporary errors

## 🔧 Improvements

### Code Quality

- Code cleanup and improved commenting
- Better error handling throughout the codebase
- Enhanced logging for debugging

### Issue Search and Review

- Added ability to search for issues with JQL
- Enhanced issue review capabilities
- Improved issue retrieval and filtering

## 🔄 API Enhancements

### Retry Configuration

HTTP requests now automatically retry with exponential backoff:
- Handles connection errors
- Manages rate limit responses (429)
- Recovers from temporary API failures

### Node.js Integration

The library now manages its own Node.js dependencies:
- Checks for `md-to-adf` package
- Automatically installs if missing
- No manual npm commands required

## ⚠️ Breaking Changes

None. All changes are backward-compatible additions.

## 🔄 Upgrade Instructions

1. Update to v0.1.1:
   ```bash
   pip install --upgrade jirakit
   ```

2. Ensure Node.js is installed:
   ```bash
   node --version  # Should show v12 or higher
   ```

3. The library will automatically install required npm packages on first use

## 🎯 Use Cases

### Improved Reliability

```python
from jirakit import JiraClient

# Connection automatically retries on failure
client = JiraClient(url, username, api_token)

# API calls retry automatically with backoff
projects = client.projects().get_all()
```

### Issue Search

```python
# Search issues by JQL
issues = client.issues().search("project=PROJ AND status=Open")
```
