# Release Notes - v0.1.0

**Release Date:** 24 February 2025
**Type:** Initial Release

## Overview

Version 0.1.0 is the initial release of jirakit, a Python library for Jira Cloud automation with template-based project deployment capabilities.

## ✨ Features

### Template-Based Project Deployment

**YAML-Driven Configuration:**
- Define complete Jira projects in YAML templates
- Automated creation of issue types, screens, workflows
- Custom field configuration
- Screen scheme and issue type scheme management

**Core Capabilities:**
- Project creation from templates
- Issue type management
- Screen and screen scheme configuration
- Workflow creation and management
- Field management and custom fields
- Group creation and management

### Project Management

```python
from jirakit import JiraClient

client = JiraClient(url, username, api_token)

# Create project from template
project = client.projects().create(
    name="My Project",
    key="PROJ",
    template=template_dict
)
```

**Issue Operations:**
- Create issues with custom fields
- Update existing issues
- Search and filter issues
- Apply workflow transitions
- Manage issue types

### Workflow Management

**Workflow Features:**
- Create custom workflows
- Define workflow statuses

### Screen Management

**Screen Configuration:**
- Create custom screens

### Field Management

### Group Management

## 🏗️ Architecture

**Markdown to ADF Conversion:**
- Converts markdown to Atlassian Document Format
- Uses `md-to-adf` Node.js package
- Supports rich text content in issues
- Handles code blocks and formatting

## 📝 Template Structure

### Example Template

```yaml
name: "Project Template"
description: "Template description"

groups:
  - name: "project-users"

issue_types:
  - name: "Task"
    description: "A task to complete"
    subtask: false

screens:
  - name: "Task Screen"
    description: "Screen for tasks"

workflows:
  - name: "Simple Workflow"
    description: "Basic workflow"
    statuses:
      - name: "To Do"
        category: "To Do"
      - name: "Done"
        category: "Done"
```

## 🔄 Usage Example

```python
from jirakit import JiraClient
import yaml

# Connect to Jira Cloud
client = JiraClient(
    url="https://your-instance.atlassian.net/",
    username="your-email@example.com",
    api_token="your-api-token"
)

# Load template
with open('template.yaml', 'r') as f:
    template = yaml.safe_load(f)

# Deploy template
project = client.projects().create(
    name="My Project",
    key="PROJ",
    template=template
)

print(f"Project created: {project.key}")
```

## ⚠️ Known Limitations

### Initial Release Constraints

- Manual Node.js package installation required
- No deployment tracking or rollback capabilities
- Limited error handling for failed deployments
- No retry logic for API failures
- Basic documentation

These limitations are addressed in subsequent releases:
- v0.1.1 adds retry/backoff and auto Node.js package installation
- v0.1.2 streamlines dependencies
