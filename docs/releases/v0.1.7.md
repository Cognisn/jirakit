# Release Notes - v0.1.7

**Release Date:** 9 October 2025
**Type:** Patch Release - Bug Fix

## Overview

This patch release fixes a critical bug caused by Atlassian deprecating the `/rest/api/3/search?jql=` endpoint. Users were experiencing 410 Gone errors when searching for issues.

## 🐛 Bug Fixes

### Fixed Deprecated API Endpoint for Issue Search

**Issue:** The Jira Cloud API deprecated and removed the GET-based search endpoint, causing 410 errors.

**Error Message:**
```
410 Client Error: Gone for url: https://*.atlassian.net/rest/api/3/search?jql=...

"The requested API has been removed. Please migrate to the /rest/api/3/search/jql API"
```

**Solution:** Migrated to the new POST-based search endpoint as per Atlassian's migration guide.

### Changes Made

- **Endpoint Migration:**
  - ❌ Old: `GET /rest/api/3/search?jql={query}`
  - ✅ New: `POST /rest/api/3/search/jql`

- **Implementation Updates:**
  - JQL query now passed in request body instead of URL parameter
  - Added `maxResults: 100` parameter to request payload
  - Added `fields: ["*all"]` parameter to ensure all fields are returned

- **Affected Method:**
  - `Issues.get_issues_updated_last_days()` in `src/jirakit/issues/__init__.py`

### Files Changed

- `src/jirakit/issues/__init__.py` - Updated search implementation
- `src/jirakit/_version.txt` - Version bump to 0.1.7

## 📚 References
